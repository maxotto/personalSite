О проекте

Предпосылки

Время идет, технологии развиваются. Предыдущий мой проект разрабатывался на свежей, в то время, версии Yii1 с вкраплениями ExtJs на фронтенде. Начинать на такой связке что-то новое в 2018 году конечно не стоит.

Поэтому, после некоторых раздумий, было принято решение изучать Angular, как основу для фронтенда.

Изучение теоретических материалов - это здорово и правильно. Умные книжки, курсы Udemy, видеоблоги и т.п. Но пока на практике не пощупаешь, что, да как, мало что в голове остается. Было принято решение оживить свой домен mikhailichenko.su в качестве учебного проекта.

Реализация

Итак, по замыслу, имеем клиентскую часть в виде SPA приложения Angular, серверную часть в виде API-сервера для предоставления данных клиентской части и админскую часть в виде классического Yii2 приложения для управления всей системой в целом.

Для старта были использованы следующие компоненты:

  • Angular CLI
  • Yii 2.0 Advanced Application Template
  • MySql

Первый вопрос, который приходится решать в более-менее серьезных проектах, - это вопрос авторизации и аутентификации. Было принято решение использовать технологию JWT с двуми токенами: основной токен и refresh-токен. Проверка логина и пароля пользователя в данном проекте осуществляется штатным способом фреймворка Yii 2. При разработке корпоративного приложения дополнительно пользователь будет проверяться на соответствие с корпоративным каталогом LDAP

Отдельные страницы, как правило, оформлены в виде модулей, подгружаемых по мере необходимости (т.н. lazyload).

Разработка ведется в IDE PhpStorm. Для локальной разработки была развернута среда Vagrant. Исходные коды хранятся на GitHub. Для управления процессом разработки и выпуска релизов используется методология "git flow".

Деплой на хостинг осуществляется c помощью npm-пакета ftp-deploy
